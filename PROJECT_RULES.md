# ELITERA PROJECT RULES v1.0

## GENERAL

Always read MASTER_CONTEXT.md before starting.

Understand the project before writing code.

Protect the current architecture.

Do not make unnecessary changes.

---

## HTML

Use semantic HTML.

Do not add unnecessary wrappers.

Reuse existing sections whenever possible.

Maintain accessibility.

---

## CSS

Reuse existing classes.

Avoid duplicate CSS.

Prefer existing variables.

Keep CSS clean.

Keep CSS modular.

Avoid large selectors.

Use clamp() when appropriate.

Respect prefers-reduced-motion.

Support focus-visible.

---

## JAVASCRIPT

Write only when necessary.

Prefer existing code.

Avoid unnecessary libraries.

Keep code lightweight.

---

## RESPONSIVE

Desktop First.

Tablet optimized.

Mobile optimized.

No broken layouts.

No horizontal scrolling.

---

## DESIGN

Keep current typography.

Keep current colors.

Keep current spacing system.

Keep current design language.

No visual regression.

---

## PERFORMANCE

Minimize CSS.

Minimize JavaScript.

Lazy-load where appropriate.

Avoid unnecessary animations.

---

## BEFORE APPLYING

Analyze.

Plan.

Show diff.

Wait for approval.

Apply changes.

Never commit automatically.

---

## QUALITY

Write maintainable code.

Write scalable code.

Write production-ready code.

Think like a Senior Front-End Architect.
